🏠 一、HTML 基础（1-25题）
第1题：<!DOCTYPE html> 的作用？
🌍 生活比喻 就像去医院前先挂号，告诉医院"我是按现代标准流程来的"。不挂号（不写 DOCTYPE），医院就用老规矩接待你，可能乱套。

📖 通俗解释 <!DOCTYPE html> 是写在 HTML 文件第一行的一句"声明"，意思是告诉浏览器：

"嘿，请你用最新的 HTML5 标准来读这个页面！"

如果不写，浏览器会进入"怪异模式"，按照上古时代 IE 浏览器的规则渲染，会导致盒子的宽高计算方式不一样，布局容易错乱。

✅ 技术要点

写了 → 标准模式（现代规范）
不写 → 怪异模式（IE 老规矩，布局会出 bug）
💡 一句话记忆法：DOCTYPE = 给浏览器的"入场券"，告诉它用哪套规则读页面。

第2题：什么是 HTML 语义化？
🌍 生活比喻 想象你在整理文件夹：

不语义化 = 所有文件都叫"文件1、文件2、文件3"
语义化 = 文件叫"合同.pdf"、"发票.jpg"、"简历.docx"
一眼就知道是什么！

📖 通俗解释 HTML 语义化就是用有意义的标签来描述内容，而不是全用 <div> 堆砌。

你想表达什么	错误写法	正确写法
页面头部	<div class="header">	<header>
导航栏	<div class="nav">	<nav>
文章内容	<div class="article">	<article>
侧边栏	<div class="aside">	<aside>
页面底部	<div class="footer">	<footer>
好处三连：

代码像说明书，别人一看就懂
搜索引擎（百度/谷歌）更容易理解你的页面，有利于 SEO
盲人用的屏幕阅读器能正确"朗读"出页面结构
💡 一句话记忆法：语义化 = 给每个区域起一个准确的名字，而不是全叫 div。

新增功能	对应标签/API	通俗理解

第4题：<meta> 标签有哪些常见属性？
🌍 生活比喻 meta 标签就像书的"版权页"——读者看不到，但出版社、图书馆、搜索引擎都会看。

📖 通俗解释 meta 标签写在 <head> 里，用户看不到它，但它告诉浏览器和搜索引擎很多重要信息：

HTML

<!-- 告诉浏览器：用 UTF-8 编码（支持中文）-->
<meta charset="UTF-8">

<!-- 告诉手机浏览器：按手机屏幕宽度显示 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 告诉搜索引擎：这个页面是关于什么的 -->
<meta name="description" content="这是一个卖手机的网站">

<!-- 告诉浏览器：用最新版本的IE内核 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">
💡 一句话记忆法：meta = 页面的"身份证背面"，给机器看的隐藏信息。

第5题：defer 和 async 的区别？
🌍 生活比喻 把 HTML 解析比作"盖房子"，JS 脚本比作"装修工"：

无属性：施工中途停工等装修工来，装修完才继续盖 🚫
async：装修工自己找时间来，一到就立刻开始装修，不管房子盖没盖好 ⚡
defer：装修工自己找时间来，等房子盖好了再按顺序装修 ✅
📖 通俗解释

text

HTML解析:  ████████---------████████  (无属性：遇到script就停)
script:              ████执行

HTML解析:  ████████████████████████   (async：不停，但script下载完就打断)
script:        ████              ██执行(随时可能)

HTML解析:  ████████████████████████   (defer：不停，等HTML解析完再执行)
script:        ████                  ██按顺序执行
使用原则：

有顺序依赖的脚本（如 jQuery 插件）→ 用 defer
完全独立的脚本（如百度统计）→ 用 async
💡 一句话记忆法：defer = 等房子盖好再装修；async = 装修工来了就立刻干，不管你准备好没。

第6题：alt 和 title 的区别？
🌍 生活比喻

alt = 图片的"备胎文字"（图挂了顶上）
title = 图片的"小纸条"（鼠标悬停才显示）
📖 通俗解释

HTML

<img src="cat.jpg" 
     alt="一只橘猫坐在阳台上"   ← 图片加载失败时显示这段文字
     title="这是我家的猫">      ← 鼠标悬停时显示的提示
重要区别：

alt 是必须写的（对盲人友好，对 SEO 有帮助）
title 可选，只是补充说明
纯装饰图片的 alt 写空字符串：alt=""（告诉屏幕阅读器"忽略我"）
💡 一句话记忆法：alt = 图片的替身演员；title = 图片的八卦小报。

第7题：<link> 和 @import 引入 CSS 的区别？
🌍 生活比喻

<link> = 提前预约好了送餐员，页面一开始加载就同步送餐 🚀
@import = 等你坐下来点完菜，才开始叫外卖，要多等一会 🐢
📖 通俗解释

HTML

<!-- link：页面加载时同时并行加载CSS，推荐！ -->
<link rel="stylesheet" href="style.css">
CSS

/* @import：在CSS文件里再去加载另一个CSS，串行，慢！ */
@import url('other.css');
5大区别：

对比	<link>	@import
加载时机	并行，快 ✅	串行，慢 ❌
写在哪里	HTML 里	CSS 里
JS 可控制	可以 ✅	不行 ❌
兼容性	全支持 ✅	IE5+才支持
推荐程度	推荐 ✅	不推荐 ❌
💡 一句话记忆法：用 link，别用 @import，因为 link 是并行加载，@import 是排队等待。

第8题：<a> 标签的 target 属性取值？
🌍 生活比喻 target = 告诉浏览器"在哪个房间打开这个链接"：

_self = 在当前房间打开（原地替换）
_blank = 开个新房间打开（新标签页）
_top = 推倒所有隔断，在最大的房间打开
📖 通俗解释

HTML

<a href="https://baidu.com" target="_self">当前页面打开</a>
<a href="https://baidu.com" target="_blank" rel="noopener noreferrer">新标签页打开</a>
⚠️ 安全提示：用 _blank 时必须加 rel="noopener noreferrer"，否则新页面可以通过 window.opener 控制你的原页面，有安全漏洞！

💡 一句话记忆法：_blank 开新标签但别忘加 rel="noopener"，否则后门没关。

第9题：行内元素和块级元素的区别？
🌍 生活比喻

块级元素 = 砖块 🧱，每块独占一行，可以控制大小
行内元素 = 文字 📝，紧跟在一行里，大小由内容决定
📖 通俗解释

text

块级元素（独占一行）：
┌─────────────────────────────┐
│ <div>我独占一整行</div>       │
├─────────────────────────────┤
│ <p>我也独占一整行</p>         │
└─────────────────────────────┘

行内元素（跟别人挤一行）：
我是<span>普通</span>文字 <a>链接</a> <strong>加粗</strong> 挤在一行里
特性	块级元素	行内元素
独占一行	✅	❌
能设宽高	✅	❌（由内容撑开）
上下 margin	✅	❌（无效）
常见例子	div, p, h1, ul, li	span, a, strong, em, img
💡 一句话记忆法：块级 = 砖块独占一行；行内 = 文字共享一行。

第10题：<iframe> 的优缺点？
🌍 生活比喻 iframe = 在你的网页里开一扇"窗户"，透过窗户看另一个网页。 优点：两边各自独立，互不打扰。 缺点：窗户本身也有重量（性能开销），通过窗户对话很麻烦（跨域通信）。

📖 通俗解释

什么时候用 iframe？

嵌入地图（百度地图、高德地图）
嵌入视频播放器（B站、优酷的分享代码）
嵌入支付页面（独立沙盒，安全）
缺点清单：

额外的 HTTP 请求，拖慢页面
搜索引擎爬虫不喜欢爬 iframe 里的内容（SEO 差）
存在"点击劫持"安全风险（需要服务器配置 X-Frame-Options 头）
和主页面通信需要用 postMessage，麻烦
💡 一句话记忆法：iframe = 网页里的网页，隔离好但沉重、跨域沟通麻烦。

第11题：data-* 自定义属性的作用？
🌍 生活比喻 data-* = 给 HTML 元素贴的"便利贴"，用来存储一些额外信息，不影响页面显示，但 JS 可以读取。

📖 通俗解释

HTML

<!-- 给商品卡片贴上"便利贴"，存储商品ID和价格 -->
<div class="product-card" 
     data-product-id="888" 
     data-price="299">
  iPhone 手机壳
</div>
JavaScript

const card = document.querySelector('.product-card');

// 读取便利贴信息（推荐方式，连字符自动转驼峰）
console.log(card.dataset.productId); // "888"
console.log(card.dataset.price);     // "299"

// 也可以用 getAttribute
console.log(card.getAttribute('data-product-id')); // "888"
💡 一句话记忆法：data-* = HTML 元素的私人小便利贴，JS 可以读写。

第12题：<form> 的 enctype 属性？
🌍 生活比喻 enctype = 表单数据打包方式，就像寄快递时选择"普通纸箱"还是"防震泡沫箱"：

📖 通俗解释

取值	通俗理解	用途
application/x-www-form-urlencoded	普通纸箱（默认）	发文本表单
multipart/form-data	防震泡沫箱（多层打包）	上传文件必须用这个
text/plain	没有包装，直接扔过去	调试用，实际不用
HTML

<!-- 普通登录表单（默认，不用写 enctype）-->
<form action="/login" method="POST">...</form>

<!-- 上传文件必须用 multipart/form-data！-->
<form action="/upload" method="POST" enctype="multipart/form-data">
  <input type="file" name="avatar">
</form>
💡 一句话记忆法：上传文件必须加 enctype="multipart/form-data"，否则文件传不过去。

第14题：<canvas> 和 <svg> 的区别？
🌍 生活比喻

canvas = 像素画板 🖼️（用马赛克拼图，放大会看到颗粒）
svg = 矢量图纸 📐（用数学公式描述图形，放多大都清晰）
📖 通俗解释

对比	Canvas	SVG
本质	像素图（位图）	数学描述（矢量）
放大效果	会模糊 😵	永远清晰 ✅
操作方式	JS 画像素点	操作 XML 节点
性能	大量图形时快	元素多了慢
适合场景	游戏、数据可视化、图片处理	图标、Logo、地图、动画
点击事件	需自己计算坐标	可直接绑定到形状
HTML

<!-- Canvas：JS 画图 -->
<canvas id="myCanvas" width="300" height="150"></canvas>
<script>
  const ctx = document.getElementById('myCanvas').getContext('2d');
  ctx.fillRect(50, 50, 100, 100); // 画一个矩形
</script>

<!-- SVG：XML 描述图形 -->
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="red"/>
</svg>
💡 一句话记忆法：Canvas 是像素画，SVG 是数学图，图标用 SVG，游戏用 Canvas。

第15题：DOM 和 BOM 的区别？
🌍 生活比喻 把浏览器比作一栋大楼：

DOM = 大楼里的房间结构（你装修操作的对象）
BOM = 大楼本身的基础设施（电梯=history，门牌=location，广播=console）
📖 通俗解释

JavaScript

// DOM（Document Object Model）= 操作页面内容
document.getElementById('btn')      // 找到页面上的按钮
document.querySelector('h1').textContent = '新标题'  // 修改标题

// BOM（Browser Object Model）= 操作浏览器本身
window.location.href = 'https://baidu.com'  // 跳转页面
window.history.back()                        // 返回上一页
window.alert('hello')                        // 弹出提示框
navigator.userAgent                          // 获取浏览器信息
关键区别：

DOM 是 W3C 统一标准，所有浏览器一致
BOM 没有统一标准，各浏览器可能有差异
💡 一句话记忆法：DOM 管页面内容，BOM 管浏览器行为，两者都挂在 window 下。

第16题：<script> 为什么建议放在 <body> 底部？
🌍 生活比喻 浏览器读 HTML 就像读书，遇到 <script> 就像遇到"请先去查字典再继续"——整个读书过程暂停，等查完字典才继续。

如果把字典放在第一页，读者就要先查字典，才能看正文，体验很差。 如果把字典放在最后，读者先读完正文，最后再看字典。

📖 通俗解释

HTML

<!-- ❌ 不好：script放在head，页面空白等JS加载完 -->
<head>
  <script src="big-file.js"></script>  <!-- 阻塞！用户看到空白页 -->
</head>

<!-- ✅ 推荐：script放在body底部 -->
<body>
  <h1>页面内容先显示</h1>
  <p>用户先看到内容...</p>
  <script src="big-file.js"></script>  <!-- 最后加载JS -->
</body>

<!-- ✅ 更现代：用 defer 属性，可以放在 head 里 -->
<head>
  <script src="big-file.js" defer></script>  <!-- 异步加载，不阻塞 -->
</head>
💡 一句话记忆法：JS 放底部或用 defer，是为了让用户先看到页面内容，不用对着空白发呆。

第17题：src 和 href 的区别？
🌍 生活比喻

src = "嵌入"，把外部资源直接塞进当前页面（像吃进肚子里）
href = "链接"，和外部资源建立关系（像牵一根绳子）
📖 通俗解释

HTML

<!-- src：把脚本"嵌入"进来，浏览器会暂停解析等它加载完 -->
<script src="app.js"></script>
<img src="photo.jpg">  <!-- 图片嵌入进来 -->
<iframe src="other.html"></iframe>

<!-- href：与样式表建立"关联"，不阻塞页面解析 -->
<link rel="stylesheet" href="style.css">
<a href="https://baidu.com">点我跳转</a>
关键区别：

src 会阻塞浏览器（等资源下载完才继续）
href 不会阻塞（同时并行加载）
💡 一句话记忆法：src = 嵌入进来（会阻塞）；href = 关联过去（不阻塞）。

第18题：图片懒加载怎么实现？
🌍 生活比喻 懒加载 = 餐厅里的"按需上菜"制度。不是一开始把所有菜全端上来，而是你走近哪桌，才给哪桌上菜。

📖 通俗解释

方式1：原生属性（最简单，推荐）

HTML

<img src="cat.jpg" loading="lazy" alt="猫咪">
加一个 loading="lazy" 就搞定了！浏览器自动处理，只有图片快进入视口时才加载。

方式2：IntersectionObserver（兼容性更好）

JavaScript

// 创建一个"观察员"，监视图片有没有进入视口
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {      // 进入视口了！
      const img = entry.target;
      img.src = img.dataset.src;     // 才真正加载图片
      observer.unobserve(img);       // 加载完就不再监视了
    }
  });
});

// 监视所有带 data-src 属性的图片
document.querySelectorAll('img[data-src]').forEach(img => observer.observe(img));
HTML

<!-- 初始不设置 src，图片不会加载 -->
<img data-src="cat.jpg" alt="猫咪" src="placeholder.gif">
💡 一句话记忆法：懒加载 = 图片滚进视野才加载，节省流量，提升首屏速度。

第19题：<table> 布局的弊端？
🌍 生活比喻 用 table 做布局就像用"餐桌"来当"书架"——虽然能放东西，但用途不对，又笨重又难调整。

📖 通俗解释

<table> 设计是用来显示数据表格的（比如成绩单、价格对比表），不是用来做网页布局的。

弊端清单：

语义错误：表格标签表示"这是数据"，不是"这是布局"
渲染慢：浏览器要等整个表格下载完才渲染（行列关联计算）
维护困难：改布局需要动表格结构，牵一发动全身
SEO 差：搜索引擎不喜欢看表格布局
响应式难做：表格不好自适应手机屏幕
✅ 正确场景：表格只用于展示真正的数据（订单列表、对比表、成绩单）。

💡 一句话记忆法：table 做布局 = 用筷子吃牛排，能用但不专业，应该用 Flex/Grid。

第20题：HTML 全局属性有哪些？
🌍 生活比喻 全局属性 = 每个 HTML 元素都可以用的"通用技能"，不管是 div、p 还是 span 都能用。

📖 通俗解释

HTML

<div 
  id="unique-id"           <!-- 唯一标识，像身份证号 -->
  class="red bold"         <!-- 样式类，像衣服款式 -->
  style="color: red;"      <!-- 行内样式，像临时换装 -->
  title="鼠标悬停显示的提示" <!-- 工具提示 -->

💡 一句话记忆法：全局属性 = 任何标签都能用的通用"装备"。

第21题：什么是 Web Worker？
🌍 生活比喻 普通 JS = 单线程，就像一个厨师又要炒菜、又要洗碗、又要点餐，一件没完不能干下一件，忙不过来就卡住了。

Web Worker = 雇了一个"后厨帮手"，帮你单独处理复杂任务（如数据计算），厨师（主线程）继续接待客人（更新 UI），互不打扰。

📖 通俗解释

JavaScript

// 主线程（main.js）
const worker = new Worker('helper.js');

// 发任务给 worker
worker.postMessage({ data: [1,2,3,4,5,...] });

// 接收 worker 计算结果
worker.onmessage = (e) => {
  console.log('计算结果：', e.data);
};
JavaScript

// helper.js（在后台线程运行）
onmessage = (e) => {
  // 做复杂计算，不卡主线程
  const result = e.data.reduce((a, b) => a + b, 0);
  postMessage(result);  // 把结果发回去
};
限制：Web Worker 不能操作 DOM（不能改页面），只能做计算。

💡 一句话记忆法：Web Worker = 开后台线程干苦力，主线程继续顺滑响应用户。

sessionStorage = 酒店的房间保险柜，退房（关标签页）就清空

第23题：HTML 可访问性（a11y）和 ARIA 属性？
🌍 生活比喻 可访问性 = 给网站装上"无障碍设施"（坡道、盲文提示、手语翻译），让残障人士也能正常使用。

text

第26题：CSS 盒模型？标准盒模型 vs IE 盒模型？
🌍 生活比喻 每个 HTML 元素都是一个快递盒：

内容（content）= 盒子里的东西
填充（padding）= 泡沫防震层
边框（border）= 纸箱本身
外边距（margin）= 盒子和其他盒子之间的距离
两种计算方式的区别：

标准盒模型：你说宽 200px，指的是"盒子里的东西"宽 200px，加上泡沫和纸箱后更宽
IE 盒模型：你说宽 200px，指的是"泡沫+纸箱+东西"加起来 200px，更直观！
📖 通俗解释

CSS

/* 标准模式（默认）：width = 内容宽度 */
.box {
  box-sizing: content-box;  /* 默认值 */
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* 实际占用宽度 = 200 + 20*2 + 5*2 = 250px，比你设的还大！ */
}

/* IE模式（推荐）：width = 内容+padding+border */
.box {
  box-sizing: border-box;  /* 推荐！ */
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  /* 实际占用宽度 = 200px，你设多少就多少，直观！ */
}

/* 全局推荐设置 */
*, *::before, *::after {
  box-sizing: border-box;
}
💡 一句话记忆法：border-box 更直观，width 就是你看到的宽度，推荐全局设置。

第27题：CSS 选择器优先级如何计算？
🌍 生活比喻 优先级 = 军衔制度，高级覆盖低级：

!important = 总司令命令（最高，不可抗）
行内样式 = 将军命令
ID 选择器 = 上校命令
类/伪类 = 中尉命令
标签 = 士兵建议
通配符 = 路人建议
📖 通俗解释

CSS

.text { color: green; }    /* 0,0,1,0 中尉 */
#title { color: red; }     /* 0,1,0,0 上校 */
/* 行内样式 style=""       1,0,0,0 将军 */
/* !important              无敌 总司令 */

<div id="title" class="text" style="color: orange;">
  我是什么颜色？ → orange（行内样式赢了）
</div>
⚠️ !important 的坏处：破坏正常优先级规则，团队项目里用多了会造成"谁能赢谁都不知道"的混乱，能不用就不用。

📖 通俗解释

CSS

第30题：position 有哪几种取值？
🌍 生活比喻

position	比喻
static	老实人，在文档流里按顺序排队
relative	在原位置基础上偏移，但原来的"坑"还在
absolute	悬浮在空中，不占位置，相对于有定位的祖先飞翔
fixed	钉在玻璃上（视口），页面滚动它不动（回到顶部按钮）
sticky	先乖乖排队，滚到某个位置后钉住（吸顶效果）
📖 通俗解释

CSS

/* relative：相对自己原来的位置偏移 */
.box {
  position: relative;
  top: 20px;   /* 从原位置向下移 20px */
  left: 30px;  /* 从原位置向右移 30px */
  /* 原来的空间仍然保留！ */
}

/* absolute：脱离文档流，相对最近的非static祖先定位 */
.tooltip {
  position: absolute;
  top: 0;
  right: 0;  /* 相对父元素右上角 */
}

/* fixed：相对视口固定（如回到顶部按钮）*/
.back-to-top {
  position: fixed;
  bottom: 20px;
  right: 20px;
}

/* sticky：吸顶导航栏 */
.nav {
  position: sticky;
  top: 0;  /* 滚动到距顶部0px时开始吸附 */
}
💡 一句话记忆法：absolute 相对祖先飞；fixed 相对视口钉；sticky 先乖后吸顶。

第31题：浮动的问题及清除方法？
🌍 生活比喻 float = 元素"飘"起来了，就像气球飞走了，地板（父元素）高度就塌了。 清除浮动 = 把气球拽回来，让地板重新撑起来。

📖 通俗解释

问题：

HTML

<div class="parent">    <!-- 高度塌陷！因为子元素浮动了 -->
  <div class="child" style="float:left">我浮了</div>
  <div class="child" style="float:left">我也浮了</div>
</div>
解决方案：

CSS

/* 方案1：伪元素清除（最推荐）*/
.parent::after {
  content: "";
  display: block;
  clear: both;
}

/* 方案2：触发BFC（overflow:hidden）*/
.parent {
  overflow: hidden;  /* 简单粗暴，但可能裁剪内容 */
}

/* 方案3：现代方案 flow-root */
.parent {
  display: flow-root;  /* 专门为清除浮动而生 */
}

/* 方案4：直接在末尾放空div（不推荐，多了无语义标签）*/
<div style="clear:both"></div>
💡 一句话记忆法：浮动导致父元素高度塌陷，用伪元素 clearfix 或 overflow:hidden 解决。

第32题：什么是 BFC？
🌍 生活比喻 BFC = 一个独立的小王国，王国内部的布局规则和外面互不干扰，边界清晰。

📖 通俗解释 BFC（Block Formatting Context，块级格式化上下文）= 一个独立的渲染区域，内部规则不影响外部。

如何触发 BFC？

CSS

/* 以下任意一种都能触发 BFC */
overflow: hidden;      /* 最常用 */
display: flex;         /* flex 容器本身就是 BFC */
display: grid;
display: inline-block;
display: flow-root;    /* 专门触发 BFC，副作用最小，推荐 */
position: absolute;
position: fixed;
float: left;
BFC 的三大用途：

CSS

/* 用途1：清除浮动 */
.parent { overflow: hidden; }

/* 用途2：防止外边距合并 */
.wrapper { overflow: hidden; }
/* 包裹其中一个有margin的元素，让它们不在同一个BFC里 */

/* 用途3：防止被浮动元素覆盖（两栏布局）*/
.main { overflow: hidden; }  /* 触发BFC，不会被左边float元素覆盖 */
💡 一句话记忆法：BFC = 独立小王国，内外互不干扰，常用来清浮动、防margin合并。

第33题：visibility:hidden、display:none、opacity:0 的区别？
🌍 生活比喻

display: none = 椅子搬走了，位置也没了
visibility: hidden = 椅子还在，用布盖住了（占着位置）
opacity: 0 = 椅子变透明了，看不见但能坐（能点击！）
📖 通俗解释

CSS

/* display:none - 完全消失，不占空间 */
.hidden { display: none; }
/* ❌ 不占空间  ❌ 不能点击  ✅ 触发重排 */

/* visibility:hidden - 隐藏但占空间 */
.invisible { visibility: hidden; }
/* ✅ 占空间  ❌ 不能点击  ✅ 只触发重绘 */

/* opacity:0 - 透明但存在 */
.transparent { opacity: 0; }
/* ✅ 占空间  ✅ 能点击！  ✅ 触发合成层 */
实际场景：

想完全隐藏且不占位 → display: none
想隐藏但保留位置（如骨架屏占位）→ visibility: hidden
想做淡入淡出动画 → opacity: 0 + transition
想透明但不响应点击 → opacity: 0 + pointer-events: none
💡 一句话记忆法：none 搬走，hidden 盖布，opacity 透明，transparency 能被点击！

第34题：外边距合并（Margin Collapse）？
🌍 生活比喻 两个人各自站岗，规定"我前面留 20cm 空间"，如果两人面对面站，不是 20+20=40cm，而是按最大的算，只留 20cm。这就是外边距合并。

📖 通俗解释

HTML

<div style="margin-bottom: 30px;">我是第一个div</div>
<div style="margin-top: 20px;">我是第二个div</div>
<!-- 两个div之间的间距 = max(30, 20) = 30px，不是50px！ -->
三种发生情况：

相邻兄弟元素的上下 margin 合并（取最大值）
父子元素：父没有 padding/border 时，子的 margin 会"穿透"到父外面
空块元素：自身上下 margin 合并
如何避免：

CSS

/* 方法1：给父元素触发 BFC */
.parent { overflow: hidden; }

/* 方法2：给父元素加 padding 或 border，隔断 margin */
.parent { padding-top: 1px; }

/* 方法3：用 flex/grid 布局（flex 子项不会 margin collapse）*/
.parent { display: flex; flex-direction: column; }
💡 一句话记忆法：垂直 margin 相遇会合并取最大值，触发 BFC 或加 padding/border 可阻止。

第35题：伪类和伪元素的区别？
🌍 生活比喻

伪类 = 元素的"情绪状态"（hover悬停、focus聚焦、active按下）
伪元素 = 元素的"虚拟子元素"（在元素前后凭空创建一个子元素）
📖 通俗解释

CSS

/* 伪类：单冒号，描述状态 */
a:hover { color: red; }          /* 鼠标悬停 */
input:focus { border: blue; }    /* 输入框聚焦 */
li:first-child { font-weight: bold; }  /* 第一个子元素 */
li:nth-child(2n) { background: gray; } /* 偶数行 */
:not(.disabled) { cursor: pointer; }  /* 排除某类 */

/* 伪元素：双冒号，创建虚拟元素 */
p::before { content: "📌 "; }   /* 在段落前插入图钉 */
p::after  { content: " ✅"; }   /* 在段落后插入勾 */
p::first-line { font-size: 18px; } /* 第一行特殊样式 */
input::placeholder { color: #ccc; } /* 输入框提示文字 */
💡 一句话记忆法：伪类=状态（:hover），伪元素=虚拟内容（::before）；单冒号vs双冒号。

第36题：CSS 水平居中（至少4种）？

CSS

第39题：em、rem、vw、vh、px 的区别？
🌍 生活比喻

px = 绝对尺寸，"这张桌子100厘米"，固定死了
em = "相对我爸的大小"（继承父元素字号）
rem = "相对老祖宗的大小"（相对根元素 html 字号）
vw = "窗户宽度的百分之几"（视口宽度）
vh = "窗户高度的百分之几"（视口高度）
📖 通俗解释

CSS

html { font-size: 16px; }  /* 根字号 */

.parent { font-size: 20px; }

.child {
  font-size: 1.5em;   /* 1.5 × 父元素(20px) = 30px，会继承叠加！ */
  font-size: 1.5rem;  /* 1.5 × 根元素(16px) = 24px，稳定！ */
  font-size: 16px;    /* 固定16px，不随缩放变化 */
  width: 50vw;        /* 视口宽度的50% */
  height: 100vh;      /* 视口全高（全屏布局常用）*/
}
使用场景建议：

px：边框、固定尺寸
rem：字号、间距（移动端适配）
%：宽度（相对父元素）
vw/vh：全屏布局、移动端适配
💡 一句话记忆法：rem 相对根元素最稳定，移动端适配首选；em 会叠加，小心使用。

第42题：z-index 如何工作？什么情况下失效？
🌍 生活比喻 z-index = 叠叠乐游戏 🎮，数字越大越在上面。但！如果不同的叠叠乐在不同的"桌子"（层叠上下文）上，那只能在各自的桌子上比较，跨桌子没意义。

📖 通俗解释

CSS

/* 正常情况：数字大的在上面 */
.a { position: relative; z-index: 1; }  /* 下面 */
.b { position: relative; z-index: 2; }  /* 上面 */

/* z-index 失效的情况 */

/* ❌ 失效原因1：没有设置 position（static不生效）*/
.c { z-index: 999; }  /* static定位，z-index无效！ */

/* ❌ 失效原因2：父元素形成了层叠上下文，子元素被限制在父级内 */
.parent { position: relative; z-index: 1; }  /* 父创建了层叠上下文 */
.child { z-index: 9999; }  /* 只在parent内部有效，赢不了parent外面的元素 */

/* ❌ 失效原因3：opacity < 1 的元素会创建层叠上下文 */
.parent { opacity: 0.99; }  /* 创建了新层叠上下文！子元素z-index被限制 */
💡 一句话记忆法：z-index 只在有 position（非static）的元素上生效；层叠上下文内部自己比较，跨上下文由父级决定。

第43题：overflow 属性有哪些取值？

hidden：把水拦住（超出部分截掉）
scroll：装个水龙头（始终显示滚动条）
auto：超出才开水龙头（按需显示滚动条）

  overflow: scroll;   /* 始终显示滚动条（有时候丑）*/
  overflow: auto;     /* 超出才显示滚动条（推荐）*/
  overflow: clip;     /* 类似hidden，但不触发BFC（新属性）*/
}

/* 单独控制方向 */

overflow-y: hidden; /* 垂直截断 */

第46题：transition 和 animation 的区别？
🌍 生活比喻

transition = 自动门：有人靠近（触发条件）才开门，开门和关门就两个状态
animation = 旋转广告牌：无需触发，自动不停旋转，可以设置多种状态
📖 通俗解释

CSS

/* transition：状态A → 状态B 的过渡，需要触发 */
.button {
  background: blue;
  transition: background 0.3s ease;  /* 0.3s 内平滑过渡 */
}
.button:hover {
  background: red;  /* hover 触发过渡 */
}

/* animation：用 @keyframes 定义多个帧，自动播放 */
@keyframes spin {
  0%   { transform: rotate(0deg); }
  50%  { transform: rotate(180deg); }  /* 可以有中间帧 */
  100% { transform: rotate(360deg); }
}

.loading-icon {
  animation: spin 1s linear infinite;  /* 无限旋转 */
}
对比	transition	animation
触发	需要状态变化（hover等）	自动播放
帧数	只有起止两帧	任意多帧
循环	不支持	支持无限循环
控制	简单	丰富（延迟、方向、次数）
💡 一句话记忆法：transition = 鼠标一碰才变；animation = 自己跳不停。

第48题：CSS 层叠（Cascading）是什么？
🌍 生活比喻 CSS = 层叠样式表，"层叠"就是多套规则叠在一起，按优先级决定"谁说了算"。

📖 通俗解释 当多个样式都想控制同一个元素时，按这个顺序决定谁赢：

text

1. !important 用户样式（最高，无法被开发者覆盖）
2. !important 开发者样式
3. 开发者普通样式  ← 我们写的CSS在这里
4. 用户自定义样式（浏览器插件等）
5. 浏览器默认样式（如a标签默认蓝色）
同级别时，选择器权重高的赢，权重相同时后写的赢（CSS后来居上）。

CSS

/* 这三条都想设置 p 的颜色 */
p { color: blue; }         /* 权重 0,0,0,1 */
.text { color: green; }    /* 权重 0,0,1,0 → 赢了！ */
p { color: red; }          /* 权重同第一条，但写在后面，赢第一条 */

/* 最终 p.text 的颜色 = green（类选择器权重更高）*/
💡 一句话记忆法：CSS = 层叠，多规则按权重+顺序决定生效，后来者居上，高权重者优先。

第49题：inherit、initial、unset、revert 的区别？
🌍 生活比喻 这四个值是 CSS 属性的"重置按钮"：

inherit = "我要跟父母一样"
initial = "我要回到出厂设置"
unset = "智能重置：能继承就继承，不能继承就出厂设置"
revert = "我要用浏览器的默认样式"
📖 通俗解释

CSS

.child {
  /* inherit：强制继承父元素的值，即使该属性默认不继承 */
  border: inherit;   /* border 默认不继承，这样强制继承父级 */
  
  /* initial：重置为 CSS 规范的初始默认值（不是浏览器默认！）*/
  display: initial;   /* display 的initial值是 inline */
  
  /* unset：有继承性的属性用inherit，没有继承性的用initial */
  color: unset;       /* color 可继承，相当于 inherit */
  border: unset;      /* border 不可继承，相当于 initial → none */
  
  /* revert：回到浏览器默认样式（如h1默认大字体）*/
  font-size: revert;  /* 回到浏览器对该元素的默认字号 */
}
💡 一句话记忆法：inherit跟父；initial出厂；unset智能重置；revert回浏览器默认。

第52题：@font-face 的作用？
🌍 生活比喻 @font-face = 从网上"进口"一种字体，让用户不用安装这个字体也能看到。就像网上点餐，厨师在你家门口现做。

📖 通俗解释

CSS

/* 定义自定义字体 */
@font-face {
  font-family: 'MyBeautifulFont';  /* 给字体起个名字 */
  src: url('font.woff2') format('woff2'),  /* 先用更小的woff2 */

  font-weight: normal;
  font-style: normal;
  font-display: swap;  /* 关键！先用系统字体显示，字体加载完再替换 */
                       /* 避免字体加载期间文字消失（FOIT问题）*/
}

/* 使用这个字体 */
body {
  font-family: 'MyBeautifulFont', '微软雅黑', sans-serif;
}
注意事项：

优先用 .woff2 格式（压缩率最高，文件最小）
记得加 font-display: swap 避免白屏
注意字体版权问题，不是所有字体都能免费商用
💡 一句话记忆法：@font-face = 进口字体，woff2格式最小，font-display:swap防白屏。

第53题：white-space、word-break、overflow-wrap 的区别？
🌍 生活比喻 这三个属性都和"文字要不要换行"有关，但控制的角度不同：

white-space = 控制"空格和回车怎么处理，要不要换行"
word-break = 控制"一个单词到边界了要不要拆开"
overflow-wrap = 控制"一个单词溢出了要不要强制换行"
📖 通俗解释

CSS

/* white-space：控制空白符和换行 */
.no-wrap { white-space: nowrap; }     /* 不换行（单行省略号用这个）*/
.preserve { white-space: pre; }       /* 保留空格和回车（代码展示）*/
.pre-wrap { white-space: pre-wrap; }  /* 保留空格，但可以换行 */

/* word-break：控制单词内部断行 */
.break-all { word-break: break-all; }  /* 任意字符处都可断行（中文排版好用）*/
.keep-all { word-break: keep-all; }    /* 中文不在词中间断，日韩文同理 */

/* overflow-wrap：只有单词整体溢出时才断行 */
.break-word { overflow-wrap: break-word; }  /* 超长单词才会被断开 */
实际场景：

中文内容换行：word-break: break-all（中文词不需要保护）
英文单词不被截断：overflow-wrap: break-word（保护单词完整性，超出才截）
单行省略号：必须加 white-space: nowrap
💡 一句话记忆法：white-space控制换不换行；word-break控制词中间断不断；overflow-wrap控制溢出时断不断。

第54题：CSS 预处理器（Sass/Less）有什么优势？
🌍 生活比喻 原生 CSS = 手工打砖块（效率低，重复劳动多） Sass/Less = 有了砖块模具和自动搅拌机（写一次模板，自动生成）

📖 通俗解释

SCSS

/* Sass 示例：体验对比 */

/* ❌ 原生CSS：重复写很多遍 */
.nav { background: #007bff; }
.nav a { color: white; }
.nav a:hover { color: #ccc; }

/* ✅ Sass：嵌套 + 变量，清爽多了 */
$primary: #007bff;  /* 定义变量 */
$white: #ffffff;

.nav {
  background: $primary;  /* 使用变量 */
  
  a {                    /* 嵌套（自动生成 .nav a）*/
    color: $white;
    &:hover { color: darken($white, 20%); }  /* & 代表父选择器 */
  }
}

/* Mixin（可复用的样式片段，像函数）*/
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.modal { @include flex-center; }  /* 使用 mixin */
.card-body { @include flex-center; }
6大优势：

变量：统一管理颜色、间距
嵌套：减少重复写选择器
Mixin：可复用样式片段（类似函数）
运算：width: 100% / 3
继承：@extend .base-style
模块化：@use 'variables'
💡 一句话记忆法：Sass/Less = CSS 的升级版，有了变量和嵌套，写起来爽多了。

第55题：pointer-events 属性？
🌍 生活比喻 pointer-events = 控制元素是否"感知"鼠标的存在。

auto（默认）= 元素能感知鼠标，click/hover 正常工作
none = 元素对鼠标"视而不见"，鼠标事件穿透到下层
📖 通俗解释

CSS

/* 常见场景：禁用按钮时不响应点击 */
.button.disabled {
  opacity: 0.5;
  pointer-events: none;  /* 灰度 + 不可点击 */
  cursor: not-allowed;
}

/* 遮罩层不阻挡下方点击 */
.decoration-overlay {
  position: absolute;
  pointer-events: none;  /* 装饰层穿透，点击事件到达下方元素 */
}

/* 加载中覆盖层 */
.loading-mask {
  pointer-events: all;   /* 阻止所有点击（防止重复提交）*/
}
💡 一句话记忆法：pointer-events:none = 点击穿透，让鼠标事件透过这个元素到达下方。

💪 三、Flexbox 布局（56-68题）
第56题：Flexbox 解决了什么问题？
🌍 生活比喻 传统布局 = 用麻将桌（table）和气球（float）来做家具摆放，笨重、难控制、居中很痛苦。

Flexbox = 有了专业的家具摆放助手，说一句话就能居中、等分、自动填充。

📖 通俗解释 Flex（弹性布局）是一维布局神器，解决了这些痛点：

CSS

/* 以前：垂直居中要写一大堆 */
.old-center {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* Flex：一行搞定垂直居中 */
.flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 以前：等高列布局几乎不可能 */
/* Flex：默认就是等高！align-items: stretch 是默认值 */
.columns { display: flex; }
💡 一句话记忆法：Flex = 现代布局神器，一维排列，居中、等分、自适应全搞定。

第57题：Flex 容器的主要属性？
📖 通俗解释

CSS

.container {
  display: flex;  /* 启动弹性布局 */
  
  /* 主轴方向：子项排列的方向 */
  flex-direction: row;           /* → 横排（默认）*/
  flex-direction: column;        /* ↓ 竖排 */
  flex-direction: row-reverse;   /* ← 横排反向 */
  flex-direction: column-reverse;/* ↑ 竖排反向 */
  
  /* 主轴对齐：子项在主轴上怎么分布 */
  justify-content: flex-start;    /* 靠左/靠上 */
  justify-content: center;        /* 居中 */
  justify-content: flex-end;      /* 靠右/靠下 */
  justify-content: space-between; /* 两端，中间均匀 */
  justify-content: space-around;  /* 每个元素两侧均匀 */
  justify-content: space-evenly;  /* 所有间距完全相等 */
  
  /* 交叉轴对齐：子项在垂直方向怎么对齐 */
  align-items: stretch;    /* 拉伸铺满（默认）*/
  align-items: center;     /* 垂直居中 */
  align-items: flex-start; /* 顶部对齐 */
  align-items: flex-end;   /* 底部对齐 */
  align-items: baseline;   /* 文字基线对齐 */
  
  /* 是否换行 */
  flex-wrap: nowrap;  /* 不换行（默认）*/
  flex-wrap: wrap;    /* 超出就换行 */
  
  /* 子项间距 */
  gap: 16px;          /* 所有方向间距 */
  gap: 8px 16px;      /* 行间距 列间距 */
}
💡 一句话记忆法：justify-content 管主轴，align-items 管交叉轴，flex-wrap 管换不换行。

第58题：Flex 子项的主要属性？
📖 通俗解释

CSS

.item {
  /* flex-grow：如何"抢"剩余空间 */
  flex-grow: 0;   /* 不抢（默认），维持原来大小 */
  flex-grow: 1;   /* 参与分配剩余空间，比例为1 */
  flex-grow: 2;   /* 比 flex-grow:1 的元素多拿一倍空间 */
  
  /* flex-shrink：空间不够时如何"让"空间 */
  flex-shrink: 1;   /* 按比例缩小（默认）*/
  flex-shrink: 0;   /* 不缩小，保持原大小（如侧边栏）*/
  
  /* flex-basis：主轴方向的初始大小 */
  flex-basis: auto;  /* 由内容决定（默认）*/
  flex-basis: 200px; /* 初始占 200px */
  flex-basis: 0%;    /* 初始为 0，从 0 开始分配剩余空间 */
  
  /* align-self：单独覆盖容器的 align-items */
  align-self: center;     /* 这个子项单独垂直居中 */
  align-self: flex-start; /* 这个子项单独顶部对齐 */
  
  /* order：视觉顺序（不改变DOM）*/
  order: 0;   /* 默认 */
  order: -1;  /* 排到最前面 */
  order: 1;   /* 排到后面 */
}
💡 一句话记忆法：grow=抢空间，shrink=让空间，basis=初始大小，align-self=个人超车。

第59题：flex: 1 是什么意思？
🌍 生活比喻 flex: 1 = "我愿意占满剩余所有空间，跟其他同样设置的兄弟平均分"。

📖 通俗解释

CSS

/* flex: 1 等于 flex: 1 1 0% */
.item { flex: 1; }
/* 等价于 */
.item {
  flex-grow: 1;    /* 可以放大 */
  flex-shrink: 1;  /* 可以缩小 */
  flex-basis: 0%;  /* 从0开始分配（不考虑内容大小）*/
}

/* 实际应用：让中间内容填满 */
.layout {
  display: flex;
  height: 100vh;
}
.sidebar { width: 240px; flex-shrink: 0; }  /* 固定宽度，不缩小 */
.main { flex: 1; }  /* 占满剩余所有宽度 */

/* 三等分 */
.col { flex: 1; }  /* 三个都是flex:1，就会三等分容器 */
💡 一句话记忆法：flex:1 = 我来填满所有剩余空间，多兄弟就平分。

第60题：flex-direction 取值？
📖 通俗解释

CSS

/* 主轴 = 子项排列的方向 */
flex-direction: row;           /* 默认，从左到右 → */
flex-direction: row-reverse;   /* 从右到左 ← */
flex-direction: column;        /* 从上到下 ↓ */
flex-direction: column-reverse;/* 从下到上 ↑ */
注意：flex-direction 改变主轴后，justify-content 和 align-items 的"方向"也跟着变！

💡 一句话记忆法：row 横排，column 竖排，reverse 反转方向。

第61题：justify-content 和 align-items 的区别？
🌍 生活比喻

justify-content = 书架上的书左右怎么放（书架长度方向）
align-items = 书在书架上高低怎么对齐（书架厚度方向）
📖 通俗解释

text

flex-direction: row 时：

主轴 → （水平）
justify-content: center  → 子项水平居中

交叉轴 ↕ （垂直）
align-items: center      → 子项垂直居中
CSS

/* 记忆技巧：justify = 主轴，align = 交叉轴 */
.center-both {
  display: flex;
  justify-content: center;  /* 主轴（水平）居中 */
  align-items: center;      /* 交叉轴（垂直）居中 */
}
💡 一句话记忆法：justify 管主轴方向，align 管交叉轴方向，方向由 flex-direction 决定。

第62题：align-content 和 align-items 的区别？
🌍 生活比喻

align-items = 一行歌手站台，每个人在自己行里怎么站位
align-content = 多行歌手站台，几行之间的上下间距怎么分配
📖 通俗解释

CSS

/* align-items：每一行内子项的交叉轴对齐（单行或多行都有效）*/
.container { align-items: center; }

/* align-content：多行之间的间距分配（只在有多行时生效）*/
.container {
  flex-wrap: wrap;           /* 必须换行！ */
  align-content: space-between; /* 行与行之间的空间分配 */
}

/* align-content 取值和 justify-content 类似 */
align-content: flex-start;    /* 所有行靠上 */
align-content: flex-end;      /* 所有行靠下 */
align-content: center;        /* 所有行垂直居中 */
align-content: space-between; /* 行间均匀分配 */
align-content: stretch;       /* 行拉伸填满（默认）*/
💡 一句话记忆法：align-items 管单行内对齐；align-content 管多行间距，必须 flex-wrap:wrap 才生效。

第63题：flex-wrap 的作用？
📖 通俗解释

CSS

/* 不换行（默认）：子项会被压缩 */
.container { flex-wrap: nowrap; }

/* 换行：子项按主轴方向排列，放不下就换行 */
.container { flex-wrap: wrap; }

/* 反向换行：第一行在底部 */
.container { flex-wrap: wrap-reverse; }
实际应用：

CSS

/* 响应式标签列表 */
.tags {
  display: flex;
  flex-wrap: wrap;  /* 标签多了自动换行 */
  gap: 8px;
}

/* 响应式卡片列表 */
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}
.card { flex: 1; min-width: 280px; }  /* 最小280px，自动换行 */
💡 一句话记忆法：flex-wrap:wrap 让子项放不下自动换行，实现简单响应式布局。

第64题：flex-grow、flex-shrink、flex-basis 分别是什么？
🌍 生活比喻 把 Flex 容器想象成一条公路，子项是车：

flex-basis = 车的"原来大小"（出发前的车长）
flex-grow = 路还有剩余空间时，车要不要"伸展"占满
flex-shrink = 路不够时，车要不要"压缩"
📖 通俗解释

CSS

/* flex-basis：初始大小（优先级高于width）*/
.item { flex-basis: 200px; }   /* 初始就是200px宽 */
.item { flex-basis: auto; }    /* 由内容决定（默认）*/
.item { flex-basis: 0; }       /* 从0开始，全靠grow分配 */

/* flex-grow：剩余空间分配比例 */
.left { flex-grow: 1; }   /* 比right少拿一倍 */
.right { flex-grow: 2; }  /* 比left多拿一倍 */

/* flex-shrink：空间不足时收缩比例 */
.sidebar { flex-shrink: 0; }  /* 不收缩，固定宽度 */
.main { flex-shrink: 1; }     /* 按比例收缩 */
💡 一句话记忆法：basis=初始大小，grow=放大比例，shrink=缩小比例。

第65题：用 Flexbox 实现"圣杯布局"？
🌍 生活比喻 圣杯布局 = 经典三栏布局：左右是固定宽度的"柱子"，中间是自适应的"圣杯"，中间内容优先渲染。

📖 通俗解释

HTML

<div class="holy-grail">
  <header>顶部导航</header>
  <div class="body">
    <nav class="left">左侧菜单（200px）</nav>
    <main class="center">中间内容（自适应）</main>
    <aside class="right">右侧栏（150px）</aside>
  </div>
  <footer>底部版权</footer>
</div>
CSS

.holy-grail {
  display: flex;
  flex-direction: column;  /* 垂直排列：header, body, footer */
  min-height: 100vh;
}

.body {
  display: flex;
  flex: 1;  /* 填满中间区域 */
}

.left {
  width: 200px;
  flex-shrink: 0;  /* 不收缩，保持200px */
}

.center {
  flex: 1;  /* 占满剩余空间 */
  min-width: 0;  /* 防止内容溢出 */
}

.right {
  width: 150px;
  flex-shrink: 0;  /* 不收缩，保持150px */
}
💡 一句话记忆法：圣杯=左右fixed+中间flex:1，用 flex-shrink:0 保护两侧不缩小。

第66题：Flex 中 float、clear、vertical-align 为什么失效？
🌍 生活比喻 Flex 容器是一个新世界，有自己的一套规则体系。float 和 vertical-align 是旧世界（文档流）的规则，进入新世界就不管用了。

📖 通俗解释

float 失效：flex 子项进入了 flex 格式化上下文，这个上下文不存在"浮动"概念。flex 自己管理子项布局，不需要 float。

clear 失效：clear 是用来清除 float 的，既然 float 都不存在了，clear 当然也没用。

vertical-align 失效：vertical-align 只对行内元素（inline/inline-block）有效，flex 子项不是行内元素，而是 "flex item"，用 align-items 和 align-self 来控制垂直对齐。

CSS

/* ❌ 这些在 flex 子项上无效 */
.flex-item {
  float: left;         /* 无效！ */
  clear: both;         /* 无效！ */
  vertical-align: middle; /* 无效！ */
}

/* ✅ 在 flex 布局中用这些替代 */
.flex-container { align-items: center; }  /* 代替 vertical-align */
.flex-item { align-self: center; }        /* 单个子项垂直居中 */
💡 一句话记忆法：flex 是独立王国，旧世界的 float 和 vertical-align 在这里不起作用。

第67题：order 属性？
📖 通俗解释

order 允许你在不改变 HTML 结构的情况下，改变 flex 子项的视觉显示顺序。

HTML

<div class="container">
  <div class="item" style="order: 3">我是第1个写的，但显示在第3位</div>
  <div class="item" style="order: 1">我是第2个写的，但显示在第1位</div>
  <div class="item" style="order: 2">我是第3个写的，但显示在第2位</div>
</div>
<!-- 显示顺序：第2个→第3个→第1个 -->
应用场景：

CSS

/* 移动端把重要内容提前展示，PC端保持原顺序 */
@media (max-width: 768px) {
  .main-content { order: -1; }  /* 移动端排第一 */
  .sidebar { order: 1; }
}
💡 一句话记忆法：order = 只改视觉顺序，不改DOM顺序，默认0，数字越小越靠前。

第68题：Flex 中让最后一个子元素靠右对齐？
📖 通俗解释

CSS

/* 方法1：给最后一个元素设置 margin-left: auto */
.nav {
  display: flex;
  align-items: center;
}

.nav .logo { /* 靠左 */ }
.nav .menu { /* 靠左 */ }
.nav .user-btn {
  margin-left: auto;  /* 把自己推到最右边！剩余空间全给margin */
}

/* 方法2：中间加一个 flex: 1 的弹性填充元素 */
.nav .spacer { flex: 1; }  /* 撑满中间，把user-btn推到右边 */

/* 方法3：用 justify-content: space-between（适合只有两端元素）*/
.nav { justify-content: space-between; }
💡 一句话记忆法：margin-left: auto 是 flex 布局里把元素推到右边的魔法咒语。

🏗️ 四、Grid 布局（69-75题）
第69题：Grid 和 Flexbox 的本质区别？
🌍 生活比喻

Flexbox = 一条流水线 🏭（一维：要么横排要么竖排）
Grid = 一张棋盘 ♟️（二维：同时控制行和列）
📖 通俗解释

text

Flex（一维）：
┌────┬────┬────┐  ← 只能控制这一行
│ A  │ B  │ C  │
└────┴────┴────┘

Grid（二维）：
┌────┬────┬────┐
│ A  │ B  │ C  │  ← 同时控制行
├────┼────┼────┤
│ D  │ E  │ F  │
├────┼────┼────┤  ← 和列
│ G  │ H  │ I  │
└────┴────┴────┘
何时选择：

导航栏、按钮组 → Flex
整页布局、卡片网格、报表 → Grid
可以混合：Grid 做整体布局，Flex 做局部细节
💡 一句话记忆法：Flex = 一维（一行或一列），Grid = 二维（行列同时控制）。

第70题：grid-template-columns/rows 和 fr 单位？
📖 通俗解释

CSS

.grid {
  display: grid;
  
  /* 定义3列 */
  grid-template-columns: 200px 1fr 200px;
  /* 左列200px，中间自适应，右列200px */
  
  /* 定义3行 */
  grid-template-rows: 60px 1fr 40px;
  /* 顶部60px，中间自适应，底部40px */
}
fr 单位（fraction = 分数）：

CSS

/* fr = 把剩余空间等分 */
grid-template-columns: 1fr 1fr 1fr;  /* 三等分 */
grid-template-columns: 1fr 2fr 1fr;  /* 中间是两边两倍宽 */
grid-template-columns: 200px 1fr;    /* 左边固定200px，右边占满剩余 */
💡 一句话记忆法：fr = 弹性单位，把剩余空间按比例分配，比 flex:1 更直观。

第71题：gap 属性的作用？
📖 通俗解释

CSS

.grid {
  display: grid;
  gap: 16px;           /* 行列间距都是16px */
  gap: 10px 20px;      /* 行间距10px，列间距20px */
  row-gap: 10px;       /* 只设行间距 */
  column-gap: 20px;    /* 只设列间距 */
}
好处：比 margin 更简洁，不需要处理第一个/最后一个元素的边距问题。

💡 一句话记忆法：gap = 格子之间的间距，Flex 和 Grid 都能用。

第72题：grid-column 和 grid-row 跨列/跨行？
🌍 生活比喻 就像合并 Excel 单元格 📊，让一个格子跨越多列/多行。

📖 通俗解释

CSS

/* 网格线编号：列线从左到右 1,2,3,4...，行线从上到下 1,2,3,4... */

.item-a {
  grid-column: 1 / 3;     /* 从第1条列线到第3条列线（跨2列）*/
  grid-row: 1 / 2;        /* 从第1行到第2行（占1行）*/
}

.item-b {
  grid-column: 1 / -1;    /* 从第1列到最后一列（铺满整行）*/
  grid-row: 1 / span 2;   /* 从第1行开始，跨2行 */
}

/* 更直观：用 span 关键字 */
.big-card {
  grid-column: span 2;    /* 跨2列（不指定从哪列开始，由 auto placement 决定）*/
  grid-row: span 3;       /* 跨3行 */
}
💡 一句话记忆法：grid-column: 1/3 = 从线1到线3；span 2 = 跨2格，像合并单元格。

第73题：repeat() 和 minmax() 函数？
📖 通俗解释

CSS

/* repeat(次数, 大小)：避免重复写 */
grid-template-columns: repeat(3, 1fr);          /* 等于 1fr 1fr 1fr */
grid-template-columns: repeat(4, 200px);         /* 等于 200px 200px 200px 200px */

/* 自动计算列数（响应式利器！）*/
grid-template-columns: repeat(auto-fill, 200px);  /* 放多少列放多少列 */
grid-template-columns: repeat(auto-fit, 200px);   /* 同上，但会拉伸填满 */

/* minmax(最小值, 最大值)：限定轨道大小范围 */
grid-template-rows: minmax(100px, auto);     /* 最小100px，内容多了自动撑高 */
grid-template-columns: minmax(150px, 1fr);   /* 最小150px，最大1fr */

/* 黄金组合：响应式卡片布局！不需要媒体查询 */
grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
/* 卡片最小240px，自动决定几列，自动换行，超级好用！ */
💡 一句话记忆法：repeat(auto-fill, minmax(240px, 1fr)) 是响应式网格的万能公式。

第74题：隐式网格和 grid-auto-rows？
🌍 生活比喻 你规划了一个3×3的棋盘（显式网格），但是棋子太多，棋盘自动扩展了（隐式网格）。隐式行的高度默认是 auto，grid-auto-rows 让你控制这些自动创建的行高度。

📖 通俗解释

CSS

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 定义3列 */
  /* 没有定义行！行是自动生成的（隐式行）*/
  
  grid-auto-rows: 200px;     /* 自动生成的行，每行200px */
  grid-auto-rows: minmax(100px, auto);  /* 最小100px，自动撑高 */
  
  grid-auto-flow: row;       /* 自动填充方向：行优先（默认）*/
  grid-auto-flow: column;    /* 列优先 */
  grid-auto-flow: dense;     /* 密集填充，尽量填满空洞 */
}

📖 通俗解释

  to   { transform: translateX(0);     opacity: 1; }
}

/* 等同于 */
@keyframes bounce {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-20px); }
  100% { transform: translateY(0); }
}

/* 第二步：应用动画 */
.modal {
  animation-name: slideIn;             /* 动画名（对应@keyframes名）*/
  animation-duration: 0.5s;           /* 持续时间 */
  animation-timing-function: ease;    /* 速度曲线 */
  animation-delay: 0s;                /* 延迟开始 */
  animation-iteration-count: 1;       /* 播放次数（infinite=无限循环）*/
  animation-direction: normal;        /* 方向（alternate=来回）*/
  animation-fill-mode: forwards;      /* 动画结束后保持最终状态 */
  animation-play-state: running;      /* 播放/暂停 */
  
  /* 简写（常用）*/
  animation: slideIn 0.5s ease forwards;

第85题：will-change 属性？
🌍 生活比喻 will-change = 预告片。提前告诉浏览器"这个元素待会要动"，浏览器提前做好准备（放到 GPU 合成层），动画就更流畅。

📖 通俗解释

CSS

/* 告诉浏览器：这个元素的 transform 和 opacity 即将改变 */
.animated-card {
  will-change: transform, opacity;
}

/* 动画触发时加上，结束后移除（用JS控制）*/
.card:hover { will-change: transform; }
/* 或者 */
element.style.willChange = 'transform';  // 即将动画
element.style.willChange = 'auto';       // 动画结束
⚠️ 注意事项（面试必说！）：

will-change 会提升合成层，占用 GPU 内存，不能滥用
不要写 will-change: all（太贪心，反而变慢）
只在需要动画的元素上用，动画结束后最好移除
提前太多加上去反而有副作用，在动画即将开始时才设置
💡 一句话记忆法：will-change = 动画预告片，让GPU提前热身，但别滥用，会占显存。

第86题：重排（Reflow）和重绘（Repaint）？
🌍 生活比喻

重排 = 重新量房、重新规划房间布局（改了尺寸，所有家具要重新摆）
重绘 = 重新刷漆（只改了颜色，布局没变，只刷漆就行）
重排 > 重绘（重排必然引起重绘，但重绘不一定重排）

📖 通俗解释

触发重排的操作（性能杀手，要减少）：

JavaScript

// 这些会触发重排：
el.style.width = '200px';      // 改宽度
el.style.display = 'block';    // 改display
document.body.appendChild(el); // 添加/删除DOM
el.offsetWidth;                 // 读取布局属性（强制刷新！）
el.scrollTop;                   // 读取滚动位置
window.getComputedStyle(el);    // 计算后样式
优化策略：

JavaScript

// ❌ 坏：循环中反复读写，每次都触发重排
for(let i = 0; i < 100; i++) {
  el.style.left = el.offsetLeft + 1 + 'px';  // 读一次写一次，循环100次
}

// ✅ 好：先读完再写
const left = el.offsetLeft;
for(let i = 0; i < 100; i++) {
  el.style.left = left + i + 'px';  // 批量写
}

// ✅ 更好：用 transform 代替修改 top/left
el.style.transform = 'translateX(100px)';  // 走合成层，不触发重排
💡 一句话记忆法：改几何属性=重排（贵），改颜色=重绘（便宜），用transform做动画最省（走GPU合成层）。

第87题：自适应正方形（宽高始终相等）？

第88题：0.5px 细线兼容方案？
📖 通俗解释 在普通屏幕上，1px 就是 1 个物理像素，没有 0.5px。但 Retina 屏（DPR=2）上，1 CSS px = 2 物理像素，所以可以通过 transform 缩放实现视觉上的 0.5px。

CSS

/* 方案1：transform scaleY 缩小（推荐）*/
.thin-line {
  height: 1px;
  background: #ccc;
  transform: scaleY(0.5);
  transform-origin: 0 0;   /* 从顶部开始缩放 */
}

/* 方案2：伪元素 + scale（常用于盒子边框）*/
.box {
  position: relative;
}
.box::after {
  content: '';
  position: absolute;
  left: 0; top: 0;
  width: 200%; height: 200%;  /* 先放大2倍 */
  border: 1px solid #ccc;
  transform: scale(0.5);       /* 再缩小到0.5 */
  transform-origin: 0 0;
}

/* 方案3：线性渐变 */
.hr {
  height: 1px;
  background: linear-gradient(to bottom, transparent 50%, #ccc 50%);
  /* 上半部分透明，下半部分有颜色，视觉上只有0.5px */
}
💡 一句话记忆法：0.5px = transform:scaleY(0.5) 或伪元素先放大再缩小。

第89题：移动端 viewport 元标签？
📖 通俗解释

HTML

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
逐项解释：

属性值	含义
width=device-width	视口宽度 = 设备屏幕宽度（默认可能是980px！）
initial-scale=1.0	初始缩放比例为1（不缩放）
maximum-scale=1.0	最大缩放为1（锁定缩放）
user-scalable=no	禁止用户双指缩放
为什么需要这个：手机浏览器默认用 980px 的"桌面视口"显示网页，所有东西都很小。加了 width=device-width 后，用设备真实宽度（如375px）显示，元素大小才正常。

💡 一句话记忆法：没有 viewport meta，手机会用 980px 显示你的页面，一切都会缩小变丑。

🌍 生活比喻 你用 CSS 画了 1px 的线，但在 Retina 屏（DPR=2）上，1 CSS px = 2 物理像素，线变成了 2 物理像素宽，看起来比设计稿粗了一倍。

    position: relative;

    position: absolute;
